# Top-down

## Узкие места и оптимизации


### ObjectPool
Для пуль и противников был реализован object pooling, что позволило избежать лишних выделений памяти и ускорить создание/удаление объектов во время игры.

### StringBuilder
Исключены аллокации при обновлении текста UI за счёт использования `StringBuilder` вместо конкатенации строк.

### Static Batching
Объединение статичных мешей для снижения количества Draw Calls.

### NavMesh
Оптимизация сетки навигации через запекание по физическим коллайдерам вместо визуальных мешей.

### Memory
Хэширование ссылок вместо получения через `GetComponent` для уменьшения накладных расходов и аллокаций.
